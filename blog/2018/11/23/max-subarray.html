连续子数组的最大和

问题描述

  描述：给定一个整数数组，找到一个具有最大和的连续子数组，返回其最大和。
  样例：给定数组[-2, 2, -3, 4, -1, 2, 1, -5, 3]，符合要求的子数组为[4, -1, 2, 1]，其最大和为6。

方法二：分治策略

  假设我们要在数组A[left…right]中寻找最大子数组。我们把数组A划分为两个规模尽量相等的子数组：A[left…mid]和A[mid+1…right]，其中 mid = (left + right) / 2。假设要寻找的最大子数组为A[i…j]，那么A[i…j]所处的位置必然是以下三种情况之一：

  • 完全位于子数组A[left…mid]中，因此 left <= i <= j <= mid；
  • 完全位于子数组A[mid+1…right]中，因此 mid < i <= j <= right；
  • 跨越了mid，因此 left <= i <= mid < j <= right。

  实际上，情况1（完全位于A[left…mid]）和情况2（完全位于A[mid+1…right]）与原问题是相同的，仍然是找最大子数组问题，只是规模变小了，所以可以递归地求解。剩下的全部工作就是寻找跨越 mid的最大子数组（情况3），然后在以上3种情况中选取和最大者。
  我们很容易在线性时间内找出跨越mid的最大子数组。此问题与原问题不同，因为它加了限制，即求出的子数组必须跨越mid。如图所示，任何跨越mid的子数组都由两个子数组A[i…mid]和A[mid+1…j]组成，其中 left <= i <= mid < j <= right。因此，我们只需找出形如A[i…mid]和A[mid+1…j]的最大子数组，然后将其合并即可。最后，方法二的时间复杂度为O(nlogn)，比方法一要高效。

方法三：kadane算法

  实际上，”连续子数组的最大和“是可以在线性时间内求解的。该问题最早于1977年提出，但是直到1984年才被Jay Kadane发现线性时间的最优解法。方法三的时间复杂度为O(n)，比方法一和方法二都快。该算法的要点如下：

  • 最大子数组不可能包含值为负数的前缀（若数组全为负数，那么假设前缀为0），例如，[-2, 1, 4]必然不可能是最大子数组，因为去掉值为负数的前缀[-2]后，可以得到一个更大的子数组[1, 4]。
  • 所以在遍历过程中，每当累加结果成为一个负值时，就应当将下一个元素作为最大子数组的起始元素，重新开始累加。
  • 由于在累加过程中，出现的最大值都会被记录，因此保证了在遍历过程中的完备性和正确性。

方法四：动态规划（Dynamic Programming）

  如果遍历到第i个元素，那么最大和要么是之前的最大和加上第i项的值，要么是第i项的值，状态方程是：DP[i] = MAX(DP[i - 1] + A[i], A[i])。从代码上来看，和kadane算法基本上是一样的，因为kadane算法也是动态规划的思想。方法四的时间复杂度也是O(n)。
